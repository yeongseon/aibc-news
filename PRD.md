## 1. 프로젝트 개요 (Project Overview)

* **프로젝트명:** AI Broadcasting Channel (AIBC)
* **서비스 정의:** AI 기반 자동 편집국이 운영하는 종합 뉴스 브리핑 채널. 매일 아침 시장·날씨·생활 등 핵심 정보를 짧고 명확하게 전달한다.
* **핵심 컨셉:**
  * 겉으로는 기존 방송사/신문사와 유사한 뉴스 채널
  * 내부적으로는 데이터 수집 → 기사 작성 → 발행까지 완전 자동화된 AI 편집국

* **차별화 포인트:**
  * 매일 같은 시간에 고정 포맷 브리핑 제공
  * 출처 기반 사실 요약 + 문장 내 맥락/인사이트 포함
  * 과장/의견 배제, 전통 언론 톤 유지

### 목표 (Goals)
1. **완전 자동화:** 데이터 수집부터 기사 발행까지 100% 자동 실행
2. **신뢰성:** 전통 언론과 유사한 톤과 형식을 유지
3. **확장성:** 뉴스, 날씨, 생활, 데이터 분석 등 종합 채널로 확장 가능
4. **수익화 준비:** 향후 광고 및 콘텐츠 확장에 적합한 구조 확보

## 4. 콘텐츠 범위 (Content Scope)

### Phase 1 (초기)
* 데일리 브리핑 (하루 1회)
  * 주요 뉴스 요약
  * 시장 동향 요약
  * 날씨 정보
  * 생활 정보

* **발행 규칙**
  * 발행 시간: 매일 06:30 KST (허용 지연 ±30분)
  * 발행 원칙: 주말/공휴일 포함 매일 발행
  * 분량: 전체 700~1,000자 내외
  * 구성: 3~5개 항목, 항목당 2~4문장
  * 각 항목에 맥락/인사이트 2~3문장 포함

### Phase 2 (확장)
* 주간/월간 요약
* 데이터 기반 인사이트 기사
* 특정 주제별 브리핑 (경제, 기술, 사회 등)

## 6. 기능 요구사항 (Functional Requirements)

### A. 에이전트 구조 (Agent Roles)

#### Agent 1: Collector
* 역할: 외부 데이터 수집
* 입력: API / 공개 데이터
* 출력: 구조화된 JSON
* 수집 범위 (Phase 1, 최소 기준)
  * 국내 주요 헤드라인 3~5건(공식 발표/보도자료/공개 지표 기반)
  * 시장 지표 2~3건(주요 지수, 환율, 원자재 중 선택)
  * 날씨 1~2건, 생활 1건 내외(합산 2~3건)
  * 각 항목의 원문 출처 URL 포함

* 출력 스키마 (초안)
```json
{
  "date": "YYYY-MM-DD",
  "items": [
    {
      "type": "headline|market|weather|lifestyle",
      "title": "...",
      "facts": ["...", "..."],
      "sources": [
        {"name": "기관/매체", "url": "https://...", "published_at": "YYYY-MM-DD"}
      ]
    }
  ]
}
```

#### Agent 2: Writer
* 역할: AI 기자
* 입력: Collector 결과 JSON
* 출력: 마크다운 기사
* 톤: 방송사 뉴스 스타일, 중립적 문장
* 본문 구조:
  * 3~5개 항목, 항목당 2~4문장
  * 항목 내 문장으로 맥락/인사이트 2~3문장 포함
  * 수치/인용 포함 시 기준시점 또는 발표일 명시

#### Agent 3: Publisher
* 역할: 편집/발행
* 기능:
  * Front Matter 자동 생성
  * Jekyll `_posts` 형식으로 변환
  * Git 커밋 및 푸시
  * 품질 검증(아래 Quality Gate 통과 시에만 발행)
  * 실패 처리/롤백
    * 수집 실패 시: 최대 2회 재시도(10분 간격)
    * 작성 실패 시: 1회 재시도, 실패 시 발행 중단 및 알림
    * 발행 실패 시: 이전 상태 유지, 실패 로그와 원인 저장
    * 오류 발생 시 당일 브리핑은 발행하지 않음(부분 발행 금지)

### B. 품질 검증 및 알림 (Quality Gate)

* 자동 발행 전 검증 기준 (모두 충족)
  * 전체 분량 700~1,000자 범위
  * 항목 수 3~5개, 각 항목 2~4문장
  * 항목당 맥락/인사이트 2~3문장 포함
  * 출처 수: 항목당 최소 1개, 전체 4개 이상
  * 동일 출처 50% 초과 금지
  * 금칙어 미포함 및 과장 표현 없음
  * 수치/인용에 기준시점 또는 발표일 포함

* 알림 기준
  * Collector/Writer/Publisher 실패 시: GitHub Actions 실패 알림
  * 품질 검증 불통과 시: 당일 발행 중단 및 실패 사유 기록

## 8. 데이터 산출물 규격 (Post Schema)

```yaml
---
layout: post
title: "[AIBC 브리핑] 오늘의 주요 이슈 요약"
author: AIBC Desk
categories: [ News ]
date: 2026-01-25
summary: "오늘의 핵심 이슈 3~5건 요약"
sources:
  - "기관/매체명 - URL"
---
```

* 본문 형식 예시
```markdown
## 오늘의 주요 이슈

1) ...
... (2~4문장, 맥락/인사이트 포함)

2) ...
... (2~4문장, 맥락/인사이트 포함)
```

* 데일리 브리핑 템플릿 (샘플)
```markdown
## 오늘의 주요 이슈

출처: 기관/매체명, YYYY-MM-DD

2) [시장 동향] ...
... (2~4문장, 맥락/인사이트 포함)
출처: 지표명/기관명, 기준시점 YYYY-MM-DD

3) [날씨] 또는 [생활] ...
... (2~4문장, 맥락/인사이트 포함)
출처: 기관명, YYYY-MM-DD
```

## 9. 운영 원칙 (Editorial Principles)

* 감정적 표현 최소화
* 사실 기반 요약
* AI 사용 사실은 About 페이지에 투명하게 명시
* 특정 개인/기업에 대한 투자 권유 금지
* 운영 원칙은 편집 철학이며, 자동 품질 규칙은 12절 가드레일을 따른다.

### 콘텐츠 스타일 가이드 (Draft)
* 제목: 24~32자, 날짜/채널명 포함(예: "[AIBC 브리핑] 1월 25일 주요 이슈")
* 문장 길이: 평균 15~25단어, 과도한 수식어 금지
* 어조: 중립적 보도체, 판단/평가 표현 금지
* 금칙어: "충격", "대박", "속보"(미확인), "확정 수익"
* 인용: 기관명/지표명/발표일 명시

## 10. 수익화 전략 (Monetization)

  * About AIBC

  * Contact
* 광고 배치 원칙
  * 상단(Header 아래) 1개
  * 본문 중간(In-article) 1개
  * 하단(Footer/추천 콘텐츠 인접) 1개
  * 모바일 UX를 해치지 않는 반응형 광고 우선
* AI 생성 콘텐츠 정책 대응
  * 출처 기반 요약 및 편집 기준을 About/Policy에 명시
  * 자동 생성 콘텐츠로 보이지 않도록 규격화된 검수 기준 유지

### 중장기 단계 (Phase 3~4)
* 뉴스레터(아침 브리핑) 광고
* 스폰서드 콘텐츠(명확한 고지 필수)
* 데이터 리포트 기반 프리미엄 콘텐츠

## 11. 정책 및 신뢰 원칙 (Policy & Trust)

### AI 활용 고지 원칙
* AIBC는 AI 기반 자동 편집국을 운영함을 About 페이지에 명확히 고지
* 단, 사용자에게는 **기존 언론사와 유사한 편집/검수 기준**을 제공

### 편집 및 정정 정책
* 사실 오류 발견 시 정정 문구 명시
* 출처가 명확하지 않은 정보는 기사화하지 않음
* 투자·의료·법률 관련 직접적인 조언 제공 금지

### 저작권/인용 정책
* 원문 복제 금지(직접 문장 재사용 최소화)
* 인용은 1~2문장 이내로 제한하고 반드시 출처 표기
* 이미지/영상은 사용하지 않음(Phase 1 기준)

### 개인정보 및 광고 정책
* Google AdSense 정책을 준수하는 개인정보처리방침 운영
* 제3자 쿠키 및 광고 네트워크 사용 고지

## 12. 콘텐츠 품질 가드레일 (Content Quality Guardrails)

* 단순 수치 나열형 콘텐츠 금지
* 모든 브리핑에는 **맥락 설명** 최소 2~3문장 포함
* 동일 템플릿 반복 사용 시에도 내용 중복 최소화
* 출처(기관, 지표, 공개 데이터) 명시 원칙
* 대량 생성 콘텐츠로 오인되지 않도록 일 발행량 제한
* 출처 수: 항목당 최소 1개, 전체 4개 이상
* 동일 출처 과다 사용 금지(전체 출처의 50% 초과 금지)
* 수치/인용 포함 시 발표일 또는 기준시점 명시

## 14. 성공 지표 (Success Metrics)

* AdSense 승인 획득
* 자동 발행 성공률 99% 이상(30일 기준, 스케줄 대비 성공 횟수)

## 16. 요약

AIBC는 **AI가 운영하지만 기존 언론처럼 신뢰를 주는 자동 뉴스 채널**을 목표로 한다. 기술은 내부에 숨기고, 사용자는 매일 반복 가능한 브리핑 경험과 안정적인 정보 소비 환경을 제공받는다.
