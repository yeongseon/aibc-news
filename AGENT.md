# AGENT.md - AIBC Newsroom Roles

AIBC는 **멀티 에이전트 편집국** 구조로 동작한다.
각 에이전트는 명확한 역할과 산출물을 가진다.

---

## 1. Collector Agent

**목표:** 외부 데이터 수집 → 구조화 JSON 생성

**입력:**
- 수집 대상 소스 (뉴스/시장/날씨/생활)
- 실행 날짜

**출력:**
- `data/collector/YYYY-MM-DD.json`

**원칙:**
- 출처 URL 포함 필수
- 동일 출처 과다 사용 금지
- 사실/공식 발표 기반 수집

---

## 2. Writer Agent

**목표:** Collector 결과를 방송사 톤 기사로 변환

**입력:**
- Collector JSON
- 템플릿/스타일 가이드

**출력:**
- 마크다운 본문
- 요약 문자열

**원칙:**
- 중립 보도체
- 과장/의견 금지
- 수치/인용 시 발표일/기준시점 명시

---

## 3. Quality Gate Agent

**목표:** 발행 전 검증

**입력:**
- 마크다운 본문
- Collector JSON

**출력:**
- `data/quality/YYYY-MM-DD.json`

**검증 항목:**
- 분량/항목 수/문장 수
- 출처 수 및 중복 비율
- 금칙어 포함 여부
- 수치 포함 시 기준시점 표기

---

## 4. Publisher Agent

**목표:** Jekyll 포스트 생성 및 발행

**입력:**
- 본문 마크다운
- 요약 및 출처

**출력:**
- `_posts/YYYY-MM-DD-aibc-briefing.md`
- 커밋/푸시

**원칙:**
- 품질 게이트 통과 시에만 발행
- 실패 시 이전 상태 유지

---

## 5. 운영 규칙

- 하루 1회 실행 (06:30 KST)
- 실패 시 자동 재시도 정책 적용
- 부분 발행 금지
